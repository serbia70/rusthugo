## Cron 命令

### 查看当前 cron
```
crontab -l
```

### 暂停同步（删除 sync-db.sh 行）
```
crontab -l | grep -v sync-db.sh | crontab -
```

###恢复同步（加回）
```
(crontab -l 2>/dev/null; echo '*/5 * * * * bash /opt/rustvps/sync-db.sh >> /var/log/db-sync.log 2>&1') | crontab -
```
### 暂停和恢复完后确认。
```
crontab -l 
```


### 查询状态
```
tail -30 /var/log/db-sync.log
```

### 在主 VPS 上
```
crontab -l | grep sync-db.sh    # 应该显示 */5
tail -f /var/log/db-sync.log    # 看同步是否正常
```

### 在备用VPS 上
```
crontab -l | grep switchback.sh # 应该显示 0 3 * * *
```

### 改时间
```
crontab -e
```

### 切换回主vps
```
bash /opt/rustvps/switchback.sh
```

### 回切完成，数据一致。现在回主 VPS 恢复 cron：
```

crontab -e
```

### 添加这行：
```

*/5 * * * * bash /opt/rustvps/sync-db.sh >> /var/log/db-sync.log 2>&1
```
### 确认：
```
tail -f /var/log/db-sync.log
```
应该看到 同步开始 → 100.71.140.70 而不是 已切备，跳过同步。
