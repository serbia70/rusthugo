查找盘符：
```
lsblk
```
假设你的 U 盘/系统盘分区为 /dev/sdb2

挂载分区：
```
mount /dev/sdb2 /mnt
```
创建 Wi-Fi 配置文件：
```
nano /mnt/etc/NetworkManager/system-connections/serbia70.nmconnection
```
文件内容（建议加入 autoconnect=true 确保开机后自动连接）：

```
[connection]
id=serbia70
type=wifi
autoconnect=true

[wifi]
mode=infrastructure
ssid=你的WiFi名称

[wifi-security]
auth-alg=open
key-mgmt=wpa-psk
psk=你的WiFi密码

[ipv4]
method=auto

[ipv6]
addr-gen-mode=stable-privacy
method=auto
```
设置安全权限（必须为 600）：
```
chmod 600 /mnt/etc/NetworkManager/system-connections/serbia70.nmconnection
```
安全卸载：
```
sync
```
```
umount /mnt
```
完成上述步骤后，将 U 盘/SD 卡插回 N1 盒子启动，盒子在加载 Armbian 系统并启动 NetworkManager 服务时，就会自动读取该文件并连接指定的 Wi-Fi


U盘写入失败，可以等N1开机后连接网线再写入

```
sudo nano /etc/NetworkManager/system-connections/serbia70.nmconnection
```
写人下面代码
```
[connection]
id=serbia70
type=wifi
autoconnect=true
autoconnect-priority=100

[wifi]
mode=infrastructure
ssid=你的WiFi名称

[wifi-security]
auth-alg=open
key-mgmt=wpa-psk
psk=你的WiFi密码

[ipv4]
method=auto

[ipv6]
addr-gen-mode=stable-privacy
method=auto
```

重新修正本地权限并加载

```
sudo chown 0:0 /mnt/etc/NetworkManager/system-connections/serbia70.nmconnection
sudo chmod 600 /etc/NetworkManager/system-connections/serbia70.nmconnection
sudo nmcli connection reload
```


再次输入
```
nmcli connection show
```
这次列表里成功出现了 serbia70,那就是成功了，可以重启试试
