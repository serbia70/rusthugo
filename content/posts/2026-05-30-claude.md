## 命令

​Claude Code 官方热门指令清单


/statusline :自定义看到的信息，模型的类型，上下文的占比，避免上下文污染

/git commit : 快照

/clear : 新对话

/init :生成claude.md


​把 Claude Code 从聊天工具，用成工程协作系统


​1. 上下文控制类

​让 Claude 不跑偏，决定它会不会越聊越乱

​/btw：像举手插一句。临时问个小问题，问完不留痕，不打断主任务。

​/compact：像会议纪要。把长对话压缩成重点，继续聊但更省上下文。

​/context：像油量表。看看当前上下文还剩多少，避免聊着聊着爆仓。

​/clear：像另起一块白板。上下文污染太重时，直接重新开始。

​/export：像归档存储。把当前对话导出为 Markdown，方便沉淀方案与复盘。

​关键区别：/compact（保留重点，继续聊） vs /clear（直接清空，重新开局）

​适合长会话、频繁插问、需要沉淀讨论过程的人。

​2. 回退与实验类

​敢试错，才敢让 Agent 大胆改代码

​/rewind：像后悔药。AI 改坏了，可以回退代码、回退对话，或只撤销其中一个。

​双击 Esc：像快捷开关。快速打开回退菜单，不用重新找历史。

​/branch：像平行宇宙。从当前节点分出一条新路线，两种方案并行试错。

​/diff：像放大镜。合并前先看清楚，Claude 到底改了什么。

​关键区别：/rewind（撤销，回到之前） vs /branch（分叉，同时探索）

​适合探索新方案、避免误改、做多版本实验的人。

​3. 质量审查类

​别让 AI 写完就直接进仓库

​/simplify：像 3 个同事并行 review。从代码复用、代码质量、运行效率三路一起挑问题。

​/review：像日常体检。对当前改动或 PR 做常规审查，先把明显问题找出来。

​/security-review：像安全审计员。专盯权限、注入、泄露、越权这些高风险点。

​/ultrareview：像深度会诊。多 Agent 联合审查，适合关键 PR 和重要上线前。

​关键区别：/review（常规 review，日常够用） vs /simplify（更像收口优化，专治冗余和不够优雅）

​适合功能开发收尾、提测前检查、关键代码上线前把关。

​4. 模型与成本控制类

​别把 Opus 当 Sonnet 烧

​/model：像手动换挡。当前任务更重推理，还是更重执行，自己选模型。

​/model opusplan：像总师做方案、工程师落地。Opus 负责想，Sonnet 负责干。

​/effort：像思考档位。简单任务别开满功率，复杂任务再提高推理强度。

​/usage /cost：像账单仪表盘。看看额度、花费和活动统计，别聊着聊着把预算烧穿。

​关键区别：全程 Opus（更强，但更贵） vs opusplan（规划强，执行省）

​适合想提升性价比、减少限额焦虑、又不想牺牲复杂任务质量的人。

​5. 自动化与远程协作类

​从聊天写代码，走向管理 Agent

​/loop：像定时巡检机器人。每隔 N 分钟自动检查一次部署、测试或日志。

​/batch：像项目经理拆活。把大任务拆成多个单元，让一组 Agent 并行推进。

​/tasks：像任务看板。查看后台 Agent 和 Bash 任务，知道谁还在跑。

​/remote-control /rc：像手机遥控器。在手机或网页端继续控制本地 Claude Code，会话完全同步。

​/install-github-app：像把 Claude 拉进 GitHub 团队。让它参与 PR、Issue 和 CI 流程。

​关键区别：/loop（重复做同一件事） vs /batch（把大事拆开并行做）

​适合盯部署、批量改造代码、远程看任务、把 Claude 接进团队流程的人。

​6. 官方热门指令分类清单（全面汇总）

​必会：/btw、/rewind、/compact、/context、/model opusplan、/simplify、/diff

​进阶：/branch、/loop、/batch、/security-review、/insights、/memory、/permissions

​协作与沉淀：/remote-control、/export、/install-github-app

​排障：/doctor、/tasks、/usage /cost

​🚀 推荐安装顺序 / 使用顺序：

​① 先学：/btw、/rewind、/compact ➔ ② 再用：/model opusplan、/simplify ➔ ③ 最后进阶到：/batch、/loop、/remote-control
