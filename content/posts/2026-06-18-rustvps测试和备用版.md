日常开发（在 testing）

# 确认在 testing 分支
git branch

# 写代码 → 提交
git add <文件>
git commit -m "描述"

# 推送到 GitHub（触发测试版部署）
git push origin testing

测试稳定后合并到 main

# 1. 在 testing 确保所有改动已提交并推送
git status          # 应该是干净的

# 2. 切到 main 并拉取最新
git checkout main
git pull origin main

# 3. 合并 testing
git merge testing

# 4. 推送（触发正式版部署）
git push origin main

 # 5. 切回 testing 继续开发
git checkout testing

快捷版（一步到位）

稳定后合并到 main 只需：

git checkout main && git pull origin main && git merge testing && git push origin main && git checkout testing

冲突处理

如果合并时冲突：

git status                    # 看哪些文件冲突
# 手动编辑冲突文件
git add <冲突文件>
git commit -m "merge testing"
git push origin main
git checkout testing



备用

# 在主 VPS 上
crontab -l | grep sync-db.sh    # 应该显示 */5
tail -f /var/log/db-sync.log    # 看同步是否正常

# 在备 VPS 上
crontab -l | grep switchback.sh # 应该显示 0 3 * * *

改时间
crontab -e


bash /opt/rustvps/switchback.sh
回切完成，数据一致。现在回主 VPS 恢复 cron：

crontab -e

添加这行：

*/5 * * * * bash /opt/rustvps/sync-db.sh >> /var/log/db-sync.log 2>&1

完了确认：

tail -f /var/log/db-sync.log



# 查看当前 cron

crontab -l

# 暂停同步（删除 sync-db.sh 行）

crontab -l | grep -v sync-db.sh | crontab -

# 恢复同步（加回）

(crontab -l 2>/dev/null; echo '*/5 * * * * bash /opt/rustvps/sync-db.sh >> /var/log/db-sync.log 2>&1') | crontab -

暂停和恢复完后 crontab -l 确认。

查询状态

tail -30 /var/log/db-sync.log
